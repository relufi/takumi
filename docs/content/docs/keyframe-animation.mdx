Takumi supports two animation workflows:

- Use `renderAnimation()` when you want a simple animated `webp`, `apng`, or `gif`.
- Use `render()` with `timeMs` when you want full control over frame rendering and external encoding tools like ffmpeg.

This is an example of a keyframe animation rendered with ffmpeg + shiki for syntax highlighting:

This guide uses the following `scene.tsx` example.

```tsx title="scene.tsx"
const stylesheets = `
  .box {
    animation: move 1000ms ease-in-out infinite alternate;
  }

  @keyframes move {
    from {
      transform: translateX(0);
    }

    to {
      transform: translateX(60px);
    }
  }
`;

export function Scene() {
  return (
    <div tw="w-full h-full items-center justify-center">
      <style>{stylesheets}</style>
      <div className="box" tw="w-10 h-10 bg-red-500" />
    </div>
  );
}
```

## `renderAnimation()`

This is the minimal API. In most cases, you should keep a single scene and animate it with CSS keyframes. The `scenes` field is still an array because you may want to compose multiple scenes into a single animated image, but Takumi does not generate transitions between them.

```tsx twoslash title="render.tsx"
/** @jsxImportSource react */
// ---cut-before---
// @noErrors
import { Renderer } from "@takumi-rs/core";
import { fromJsx } from "@takumi-rs/helpers/jsx";
import { Scene } from "./scene";

const renderer = new Renderer();

const { node, stylesheets } = await fromJsx(
  <Scene />,
);

const output = await renderer.renderAnimation({
  width: 100,
  height: 100,
  fps: 30,
  format: "webp",
  stylesheets,
  scenes: [
    {
      durationMs: 1000,
      node,
    },
  ],
});
```

## `render()` + ffmpeg

The [`ffmpeg-keyframe-animation` example](https://github.com/kane50613/takumi/tree/main/example/ffmpeg-keyframe-animation) renders raw frames with `render()` and streams them into ffmpeg. This is the better route when you want video output or tighter control over the pipeline.

The core idea is:

1. Build the scene once.
2. Render each frame at a specific `timeMs`.
3. Pipe the raw RGBA frames into ffmpeg.

```tsx twoslash title="render.tsx"
/** @jsxImportSource react */
// ---cut-before---
// @noErrors
import { Renderer } from "@takumi-rs/core";
import { fromJsx } from "@takumi-rs/helpers/jsx";
import { spawn } from "bun";

const renderer = new Renderer();
const fps = 30;
const durationSeconds = 4;
const width = 1200;
const height = 630;
const totalFrames = fps * durationSeconds;

const { node, stylesheets } = await fromJsx(<Scene />);

const ffmpeg = spawn(
  [
    "ffmpeg",
    "-y",
    "-f",
    "rawvideo",
    "-pixel_format",
    "rgba",
    "-video_size",
    `${width}x${height}`,
    "-framerate",
    `${fps}`,
    "-i",
    "pipe:0",
    "output.mp4",
  ],
  { stdin: "pipe", stdout: "ignore", stderr: "ignore" },
);

for (let frameIndex = 0; frameIndex < totalFrames; frameIndex++) {
  const timeMs = (frameIndex / fps) * 1000;
  const frame = await renderer.render(node, {
    width,
    height,
    format: "raw",
    stylesheets,
    timeMs,
  });

  ffmpeg.stdin.write(frame);
}

ffmpeg.stdin.end();
await ffmpeg.exited;
```
